# Context Bootstrap Protocol

**Статус:** Canonical  
**Версия:** v1.0

## Назначение

Настоящий протокол определяет **обязательный порядок инициализации контекста новой сессии (чата)** между пользователем и AI-ассистентом.

Цель протокола:
- исключить потерю контекста между сессиями;
- обеспечить воспроизводимость работы;
- зафиксировать минимальный и расширенный набор артефактов, необходимых для старта работы;
- исключить неявные предположения и «память чата» как источник истины.

---

## 1. Область применения

Протокол применяется:
- при начале **каждой новой сессии**;
- при возобновлении работы после паузы;
- при переходе в новый чат в рамках одного проекта.

Протокол обязателен к применению до начала любой содержательной работы.

---

## 2. Базовые принципы bootstrap

1. **Документы имеют приоритет над контекстом чата.**
2. **Отсутствующий документ считается несуществующим.**
3. **AI не делает предположений о состоянии проекта без артефактов.**
4. **Все интерпретации выполняются в соответствии с Canonical Artifact Registry.**

---

## 3. Минимальный Bootstrap-пакет (обязательный)

Следующие артефакты **обязательны** для старта любой новой сессии:

1. `project_summary_vX.Y.md`  
2. `session_closure_log_YYYY_MM_DD_vX.Y.md` (последний)  
3. `project_history_log.md`

Без предоставления минимального bootstrap-пакета:
- AI обязан запросить недостающие документы;
- либо явно зафиксировать ограниченный режим работы.

---

## 4. Расширенный Bootstrap-пакет (по ситуации)

Следующие артефакты предоставляются **при необходимости**, в зависимости от задач сессии:

- `techspec_vX.Y.md` — при обсуждении архитектуры и требований;
- `plan_vX.Y.md` — при поэтапной реализации;
- server summary — если затрагивается инфраструктура.

---

## 5. Governance-артефакты

Следующие документы считаются **постоянно действующими**, если явно не указано иное:

- `canonical_artifact_registry.md`
- `context_bootstrap_protocol.md`
- `session_closure_protocol.md`
- `session_finalization_playbook.md`
- `documentation_rules_style_guide.md`
- `versioning_policy.md`

Повторное прикладывание governance-артефактов не требуется, если их версия не изменялась.

---

## 6. Порядок интерпретации документов

AI-ассистент обязан интерпретировать артефакты в следующем порядке приоритета:

1. Canonical Artifact Registry  
2. Governance-артефакты  
3. State-артефакты  
4. Design-артефакты

При конфликте интерпретаций применяется данный порядок.

---

## 7. Ограничения и отказ от ответственности

Если bootstrap-пакет:
- неполный,
- противоречивый,
- содержит устаревшие версии,

AI обязан:
- явно указать на проблему;
- приостановить принятие архитектурных решений;
- запросить корректные артефакты.
