# PDF Extractor — RU Blocks Extraction Policy v1.0

**Project:** PDF Extractor  
**Phase:** 2.4 BoundaryDetector  
**Status:** Canonical  
**Version:** v1.0  
**Scope:** Детерминированные правила построения anchors типов `ru_*` из `text_block` (Anchors Contract v1.0)

---

## 1. Цель

Зафиксировать воспроизводимый алгоритм выделения required RU blocks:
- `ru_title`
- `ru_authors`
- `ru_affiliations`
- `ru_address`
- `ru_abstract`

из массива `text_block` anchors, который генерирует MetadataExtractor.

Документ является нормативным контрактом: любые изменения — только через v1.1+.

---

## 2. Входные данные

Вход: anchors типа `text_block` со следующими полями:
- `page` (int)
- `text` (string)
- `lang` (`ru|en|unknown`)
- `bbox` (float[4])
- `font_size` (number)

---

## 3. Нормализация текста (детерминированно)

Для каждого `text_block` вычисляется:
- `text_norm` = исходный `text` с:
  - заменой множественных пробелов на один пробел
  - `strip()` по краям

Дополнительная лингвистическая обработка запрещена.

---

## 4. Группировка строк в кандидаты блоков

Группировка выполняется строго в пределах одной страницы.

### 4.1 Условие “слияния соседних blocks”
Два соседних `text_block` (на одной странице) объединяются в один кандидат, если одновременно выполнено:
- оба `lang == ru`
- `abs(font_size_1 - font_size_2) <= 0.5`
- вертикальная близость: `abs(y0_next - y1_prev) <= 6.0`  
  где `y0_next` — верх bbox следующего блока, `y1_prev` — низ bbox предыдущего

Текст объединяется через пробел.  
BBox кандидата: min/max по координатам.

---

## 5. Определение ru_title (required)

На каждой странице выбирается кандидат `ru_title`, если:
- `lang == ru`
- находится в “верхней зоне”: `bbox_mid_y <= 0.40 * page_height`  
  где `bbox_mid_y=(y0+y1)/2`
- имеет максимальный `font_size` среди ru-кандидатов страницы
- длина текста: `len(text_norm) >= 10`

Если условий нет — `ru_title` на странице не создаётся.

---

## 6. Определение ru_abstract (required)

`ru_abstract` выбирается как ru-кандидат, который:
- содержит маркер (case-insensitive, ru): `Аннотация` в начале строки
- текст блока включает всё после маркера

Если маркер не найден — `ru_abstract` не создаётся.

---

## 7. Определение ru_authors (required)

`ru_authors` выбирается как первый ru-кандидат после `ru_title` (по y0), который удовлетворяет:
- длина >= 5
- содержит как минимум один разделитель списка персон: запятая `,`
  ИЛИ паттерн инициалов: `\b[А-ЯЁ]\.\s?[А-ЯЁ]\.`

Если не найден — `ru_authors` не создаётся.

---

## 8. Определение ru_affiliations и ru_address (required)

После `ru_authors` выбираются следующие ru-кандидаты по y0:

- `ru_affiliations`: первый кандидат, содержащий маркеры учреждений (regex, ru, case-insensitive):  
  `университет|институт|больниц|центр|кафедр|факультет|академ`

- `ru_address`: первый кандидат, содержащий маркеры адреса (regex, case-insensitive):  
  `\bг\.\s|ул\.\s|пр\.\s|д\.\s|Россия|РФ|Москва|Санкт-Петербург`

Если не найдено — соответствующий блок не создаётся.

---

## 9. Выход

Для каждого найденного блока создаётся новый anchor:
- `type`: `ru_title|ru_authors|ru_affiliations|ru_address|ru_abstract`
- `text`: text_norm
- `lang`: `ru`
- `bbox`: bbox кандидата
- `page`: страница кандидата
- `confidence`: фиксированно `0.90`

---

## 10. Ограничения

- Никакой ML.
- Никаких вероятностных скорингов.
- Никаких “пониманий смысла” кроме проверок маркеров и regex, перечисленных в документе.
