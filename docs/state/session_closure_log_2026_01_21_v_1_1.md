2) Исправлен **stdin-контракт** BoundaryDetector: поддержка входного envelope-формата от MetadataExtractor (`payload = raw.get("data", raw)`), устранён `exit_code=10` из-за ожидания полей на верхнем уровне.
3) Уточнён RU/EN дедуп-алгоритм (цель: убрать EN-дубли стартов страниц при сохранении RU-стартов).

5) Выполнен прогон acceptance-check: `exit_code=0`, `Exact match: True`, `count=28`.
6) Обнаружена эксплуатационная проблема: при запуске в Claude Code с использованием pipe (`cat ... | python ...`) JSON-результат оказывался в `stderr`, а `stdout` становился пустым.
7) Проведена диагностика вывода и выполнен патч эмиттера: вывод JSON сделан через запись в fd1 (`os.write(1, ...)`). После этого acceptance-прогон подтверждён **при запуске с input redirection** (`python detector.py < anchors.json`): stdout содержит JSON, stderr пуст.

2) Для `mg_2025_12` канонический список стартов (28) принят как regression-oracle (golden).
3) Изменения, предложенные Claude Code по stdout/stderr (fd1 write) — **подлежат коммиту** (приоритет перед завершением сессии).
